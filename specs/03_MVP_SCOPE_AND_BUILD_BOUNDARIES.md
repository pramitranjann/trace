# 03 — MVP Scope and Build Boundaries

## MVP Goal

Ship a complete, small product experience that proves the See / Hear / Produce model.

## In Scope

### Letters

Only:

- A — Apple
- B — Ball
- C — Cat
- D — Dog
- E — Elephant

### Modes

All three modes are in scope:

- See
- Hear
- Produce

### See

- letter recognition
- word recognition
- OCR scan challenge
- multiple-choice answers

### Hear

- letter sound match
- word sound match
- pronunciation playback using Web Speech API

### Produce

- air tracing with MediaPipe
- basic path comparison
- paper writing capture
- forgiving feedback

### Platform

- phone
- desktop
- Vercel deployment

## Out of Scope

Do not build:

- user accounts
- authentication
- database
- backend
- parent dashboard
- admin dashboard
- streaks
- XP system
- achievements
- payments
- subscription
- multi-language content
- Hindi content
- more than A-E
- speech recognition
- AI tutoring
- complex handwriting ML
- real teacher/classroom management

## MVP Quality Bar

The product should be:

- usable
- playful
- responsive
- demoable
- deployable

It does not need to be:

- scalable
- complete
- highly personalized
- production monetizable

## Important Boundary

The MVP can be technically simple but must feel like a real product direction, not a raw webcam demo.
