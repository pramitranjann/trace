# 06 — UX Principles Before UI Import

This file guides implementation before the final UI is designed or imported.

## Product Feel

Trace should feel like a toy that teaches, not a course that grades.

The user should feel:

"I want to try again."

Not:

"I failed the exercise."

## Core UX Rules

### One Task Per Screen

Do not crowd the screen with:

- menus
- dashboards
- settings
- multiple panels
- complex progress systems

### Show, Do Not Tell

When possible, use:

1. animation
2. gesture
3. sound
4. illustration
5. text

Text should be the fallback.

### Make Feedback Immediate

Every tap, trace, answer, and capture should produce visible feedback.

### Be Forgiving

The product is for learning.
The MVP should not punish imperfect writing or movement.

### Keep It Fast

A challenge should usually take less than 20 seconds.

A Quick Lesson should take under 60 seconds for one letter.

## UX Patterns

### Buttons

Large, rounded, bottom-friendly on mobile.

### Cards

Bright, solid, simple, highly tappable.

### Camera Screens

Camera should dominate the screen.
Controls should be minimal.

### Feedback

Use playful animation:
- bounce
- confetti
- glow
- particles
- sound

## Temporary UI Rule

Temporary UI is allowed, but must be:

- clean
- playful
- responsive
- easy to replace

Do not build a generic dashboard just because it is faster.
